Convex Decomposition of a Polygon

A polygon is a closed shape made up of straight line segments connected end-to-end. Convex decomposition is the process of breaking down a polygon into smaller convex polygons.

What is a Convex Polygon?

A convex polygon is a polygon in which all interior angles are less than or equal to 180 degrees. In other words, the polygon does not have any "dents" or concave angles.

Why Perform Convex Decomposition?

Convex decomposition can be useful for various purposes, including:

  • Simplifying a complex polygon into smaller, more manageable shapes
  • Creating collision detection algorithms for video games and simulations
  • Optimizing rendering performance in computer graphics

How to Perform Convex Decomposition

This report shows the convex decomposition of a polygon. The polygon is defined by a set of vertices. The convex decomposition is computed in accordence of the following research paper.

Data structures used in the implementation are:

  • DCEL (Doubly Connected Edge List)
  • Vector

Input:

The input is a polygon defined by a set of vertices. The vertices are defined in a clockwise order.

Output:

The output is a set of decomposed convex polygons in the clockwise order.

Functions used in our code:
1. Bool Angles

2. Bool Notch

3. Int Side

4. Vector Rectangle

5. Vector of Vector Decompose

Explanation

1. Angles:

It is used to check whether the interior angle of a given point is less than 180 degrees or not.

Inputs: Point A, Point B, Point C.

Output: Boolean value. True if the angle at Point B is less than 180 degrees and false otherwise.

2. Notch:

It is used to check whether a given point is a notch or not. (Note that a notch is a point which has interior angle greater than 180 degrees.)

Inputs: Point A, Point B, Point C.

Output: Boolean value. True if the point Point B is a notch, False otherwise.

3. Side:

It is used to check whether a given point is on the left side or right side of a line segment.

Inputs: Point A, Point B, Point C.

Output: Integer value. 1 if the point Point C is on the left side of the line segment AB, -1 if the point Point C is on the right side of the line segment AB, 0 if the point Point C is on the line segment AB.

4.Rectangle

This function is used to check if the notches found in the given polygon lie inside the smallest possible rectangle ( parallel to the axes) enclosing the smaller convex polygon

Inputs: The convex polygon obtained after decomposition in nth iteration

Output: Coordinates of the smallest possible rectangle that encloses the given polygon

5. Decompose

This code is implementing a polygon partitioning algorithm. The algorithm takes a polygon with a set of vertices, and then finds the convex polygon from the set of vertices. It then splits the polygon into convex polygons by finding notches.
